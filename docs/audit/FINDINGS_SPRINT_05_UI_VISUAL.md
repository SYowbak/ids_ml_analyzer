# Висновки аудиту — Спринт 05 (Візуальна та UX-перевірка)

Дата: 2026-04-09
Обсяг перевірки:
- `src/ui/app.py`
- вкладки `home`, `training`, `scanning`, `models`, `history`
- headless capture артефакти

## Підсумок спринту
UI функціонально покриває операційні задачі, але має нерівномірну стабільність в headless-режимі та перевантажені інформаційні блоки у ключових сценаріях.

## Знахідки

### F-05-01: Нестабільний hydration/ready-state для headless візуальної перевірки
- Severity: HIGH
- Вплив: автоматизовані screenshot/DOM-перевірки можуть фіксувати проміжний стан.
- Рішення: формальний ready-marker і стабілізація часу рендеру для CI-кадрів.
- Поточний стан: ВІДКРИТО.

### F-05-02: Надлишкова щільність сигналів у scan output
- Severity: MEDIUM
- Вплив: оператору складніше швидко зчитати основне рішення та причину.
- Рішення: чітке групування: verdict, confidence/score, рекомендації, технічні деталі.
- Поточний стан: ВІДКРИТО.

### F-05-03: Неповна уніфікація термінів між вкладками
- Severity: LOW
- Вплив: локальна когнітивна плутанина при переході training <-> scanning <-> history.
- Рішення: єдиний словник термінів у UI.
- Поточний стан: ВІДКРИТО.

## Висновок Sprint 05
UI придатний до роботи, але потребує UX-hardening для стабільних візуальних регресій і кращої операційної читабельності.
