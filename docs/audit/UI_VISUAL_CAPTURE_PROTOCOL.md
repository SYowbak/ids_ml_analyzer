# Протокол візуального UI-capture

Дата: 2026-04-09
Мета: отримувати відтворювані артефакти стану UI для аудиту та регресій.

## 1. Підготовка середовища
1. Запустити застосунок у headless режимі (`streamlit run ... --server.headless true`).
2. Дочекатися стабільного старту сервера.
3. Використовувати фіксовану роздільність capture (напр., 1366x768).

## 2. Порядок захоплення екранів
1. Головна вкладка (базовий стан).
2. Тренування (до запуску і після завершення).
3. Сканування (до обробки і після отримання verdict).
4. Моделі (список артефактів).
5. Історія (щонайменше один свіжий запис).

## 3. Мінімальний набір артефактів
- screenshot кожної вкладки;
- DOM dump ключових контейнерів;
- коротка примітка про readiness-стан (чи були loading/skeleton залишки).

## 4. Правила якості capture
- Не фіксувати кадри під час активного перерендеру.
- Для нестабільних блоків робити повторний кадр після короткої паузи.
- Окремо позначати, якщо дані залежать від зовнішніх артефактів (моделі, датасети).

## 5. Що порівнювати між релізами
- Наявність/відсутність ключових UI-блоків.
- Зміни текстів verdict/score/severity.
- Зміни в структурі таблиці історії.
- Будь-які нові блокуючі попередження або помилки в інтерфейсі.

## 6. Репортинг
Кожен capture-прохід завершується коротким звітом:
- що перевірено;
- які відхилення знайдено;
- що є критичним для операторського рішення;
- які рекомендації на наступний цикл.
