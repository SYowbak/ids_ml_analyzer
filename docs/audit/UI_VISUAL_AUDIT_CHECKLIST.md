# Чекліст візуального аудиту UI

Дата: 2026-04-09
Мета: перевіряти UI не «на вигляд», а за повторюваними критеріями, які впливають на операційну придатність.

## Глобальні критерії
- [ ] Додаток стабільно відкривається в headless режимі.
- [ ] Немає нескінченних skeleton/loading станів.
- [ ] Контент у ключових блоках не обрізається на типових роздільностях.
- [ ] Ключові сценарії доступні без ручного refresh.

## Вкладка Головна
- [ ] Основні метрики/статуси відображаються без помилок.
- [ ] Пояснення стану системи коротке та зрозуміле.

## Вкладка Тренування
- [ ] Елементи вибору датасету і параметрів читабельні.
- [ ] Прогрес навчання не блокує UI в неконтрольований спосіб.
- [ ] Після навчання метрики/артефакти відображаються консистентно.

## Вкладка Сканування
- [ ] Обрана модель і тип вхідного файлу візуально очевидні.
- [ ] Verdict/score/severity не суперечать одне одному в подачі.
- [ ] Статус збереження в історію (`history_saved`) відображається явно.

## Вкладка Моделі
- [ ] Список моделей повний і стабільний після оновлення сторінки.
- [ ] Ключова metadata доступна без технічних «сирих» полів.

## Вкладка Історія
- [ ] Новий scan-запис з'являється без ручної діагностики.
- [ ] Основні поля історії коректно форматовані.

## Фіксація дефектів
Для кожного дефекту додавати:
1. Кроки відтворення.
2. Фактичний результат.
3. Очікуваний результат.
4. Screenshot або DOM-фрагмент.
5. Ризик для операторського рішення.
