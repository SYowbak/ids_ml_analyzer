# Обмеження на обробку даних у системі (Для тестування)

Зверніть увагу: для пришвидшення циклів розробки та тестування системи у коді застосовано кілька обмежень на зчитування даних. Якщо ви плануєте використовувати систему в Production-середовищі (для обробки повних масивів сирих даних), ці ліміти варто зняти або збільшити.

## 1. Обмеження PCAP файлів
У конфігураційному файлі `config/adaptive_config.yaml` встановлено ліміт на кількість пакетів:
- Файл: `config/adaptive_config.yaml`
- Параметр: `max_packets: 100000`
- Вплив: Парсер зупиниться після 100 000 пакетів для кожного pcap файлу.

## 2. Обмеження при завантаженні CSV файлів (Data Loader)
У логіці завантаження даних іноді застосовують обмеження `max_rows` для запобігання переповненню пам'яті:
- Файл: `src/core/data_loader.py`
- Рядки: ~104 та ~131
- Вплив: `pd.read_csv` використовує аргумент `nrows=max_rows`.

## 3. Обмеження для візуалізації в UI
Для швидкого відтворення графіків у браузері застосовано семплювання (випадковий відбір):
- Файл: `src/ui/scan_renderer.py`
- Перемінна обмеження: `max_rows` (значення ~80 000 або ~18 000 залежно від режиму) та функції на кшталт `anomalies.head(25)`.
- Функція: `_sample_for_visualization` (`df.sample(...)`)
- Файл: `src/services/visualizer.py`
- Перемінна обмеження: `sample_limit: int = 20000`

## 4. Тренування моделей
При зчитуванні прев'ю датасетів для таблиць в інтерфейсі завантажується лише по 5 рядків:
- Файл: `src/ui/tabs/training.py`
- Аргумент: `nrows=5` під час читання прев'ю датасета.
